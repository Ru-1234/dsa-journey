# MrBeast Challenge Route Manager (Question-01)

A custom C++ simulation of a survival-based challenge facility inspired by large-scale elimination games.

The system uses:
- Doubly Linked Lists for challenge route management
- Circular Linked Lists for player rotations inside zones

## Features
- Dynamic zone insertion by difficulty
- Circular player arena system
- Mini-challenge simulation
- Player elimination handling
- Zone completion and progression
- Locked/unlocked zone mechanics
- Recovery mode for low player count
- Automatic route reorganization
- Zone removal conditions
- Interactive menu-driven interface

## Data Structures Used
- Doubly Linked List
- Circular Linked List
- Dynamic Memory Allocation

# Circle of Betrayal - Contestant Arena Simulation (Question-02)

This system simulates a dynamic circular contestant arena used in a reality competition environment.

## Key Features:
- Doubly circular linked list for contestant management
- Circular singly linked list for alliance groups
- Real-time betrayal and elimination system
- Trust-based interaction mechanics
- Alliance formation, collapse, and merging
- Migration and repositioning based on trust score
- Dynamic structural updates of circular relationships

## Core Data Structures Used:
- Circular Doubly Linked List (Contestant Circle)
- Circular Singly Linked List (Alliance Groups)

## Constraints
- No STL containers for core structures
- Pointer-based implementation
- Fully menu-driven program
