# Emergency Response Engine (Question-01)

Custom Stack and Queue based emergency incident management system developed for Data Structures Assignment 2.

## Features

- Emergency case registration using FIFO queues
- Incident dispatch and resolution handling
- K-group queue reversal using stacks
- Symmetry audit for severity pattern analysis
- Severity-based queue segregation
- Dynamic memory management without STL
- Menu-driven console interface

## Concepts Used

- Queue implementation
- Stack implementation
- Pointer manipulation
- Dynamic memory allocation
- Queue reversal algorithms
- Palindrome/symmetry checking

## Constraints

- No STL containers
- No arrays for temporary storage
- No linked list libraries
- Custom Stack and Queue implementations only

# Transaction Orchestration Engine (Question-02)

A custom-built C++ simulation of a deterministic financial transaction processing system using only manually implemented stacks, queues, circular queues, and deque structures.

## Features
- Queue using two stacks
- Stack using queues
- Circular queue for suspicious transactions
- Infix to postfix fee evaluation
- Rollback and reverse compensation
- Priority escalation handling
- Sliding window fraud analytics
- Load shedding simulation
- Deterministic transaction replay concepts

## Constraints
- No STL containers
- No recursion
- No global buffers
- Pointer-based custom implementations only
